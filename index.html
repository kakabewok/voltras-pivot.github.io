Create Virtual Account Payment

http://172.16.10.51:31015/router

Membuat Virtual Account (VA) untuk pembayaran pada bank yang dituju. Mengembalikan nomor VA beserta detail akun dan waktu kedaluwarsa.

POST · Notification Callback

Payment Notification Callback

{Payment Callback URL} — URL dikonfigurasi per domain/client

Setelah transaksi Virtual Account diproses, Payment Gateway mengirimkan notifikasi ke Payment Callback URL milik client. Client wajib memvalidasi transaksi dan mengembalikan true (berhasil) atau false (gagal/tidak dikenal).

POST {Payment Callback URL}
Arah request: Payment Gateway → Client.  ·  Endpoint ini diimplementasikan dan di-host oleh client, bukan oleh Payment Gateway.

Alur Notifikasi

CustomerMelakukan Pembayaran VA
→
BankKonfirmasi ke Gateway
→
Payment GatewayPOST ke Payment Callback URL
→
ClientValidasi & balas true/false
Jika client tidak dapat dijangkau atau melempar exception, Payment Gateway mencatat status gagal notifikasi dan tidak melakukan retry otomatis — pastikan Payment Callback URL selalu tersedia.

Request Header (dikirim oleh Payment Gateway)

Header Value Keterangan
v-api-version {implementationVersion} Versi implementasi NotificationService saat runtime. Client dapat menggunakan nilai ini untuk verifikasi kompatibilitas.
domain {domain} Identifier domain/client pemilik notifikasi ini. Gunakan untuk memvalidasi bahwa notifikasi ditujukan ke domain yang benar.

Request Parameters

# Parameter Type Keterangan Contoh
1 transactionId required string ID unik transaksi. Sesuai dengan orderId pada sisi Payment Gateway. "TRX-20260609-001"
2 amount required double Nominal yang dibayarkan customer dalam Rupiah. Client wajib memvalidasi nilai ini sesuai dengan order yang tersimpan. 10000.0

Contoh Request yang Diterima Client (cURL equivalent)

curl --location --request POST 'https://client.example.com/notify' \
--header 'v-api-version: 1.1.5' \
--header 'domain: sandbox' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'transactionId=TRX-20260609-001' \
--data-urlencode 'amount=10000.0'

Response yang Diharapkan dari Client

Value Type Kondisi
true Boolean Notifikasi diterima dan diproses sukses. Transaksi valid, amount cocok, dan status order berhasil diperbarui di sisi client.
false Boolean Notifikasi gagal diproses. Transaksi tidak ditemukan, amount tidak sesuai, atau terjadi error internal di sisi client.
Jika client melempar exception TransactionNotFound atau exception lainnya, Payment Gateway menangkap exception tersebut dan mencatat hasil notifikasi sebagai false.

HTTP Status Code (dari sisi client)

Code Status Keterangan
200 OK Request diterima. Body berisi true atau false.
404 / Exception TransactionNotFound Transaksi tidak ditemukan. Gateway mencatat notifikasi sebagai gagal.
400 Bad Request Parameter tidak lengkap atau format tidak valid.
500 Internal Server Error Kesalahan internal di sisi client. Gateway mencatat notifikasi sebagai gagal.

Panduan Implementasi untuk Client

Langkah Deskripsi
1. Verifikasi header domain Pastikan nilai header domain sesuai dengan domain yang terdaftar, untuk mencegah notifikasi palsu dari pihak lain.
2. Cari transaksi Cari order berdasarkan transactionId. Jika tidak ditemukan, lempar TransactionNotFound atau kembalikan false.
3. Validasi amount Bandingkan nilai amount dari notifikasi dengan nilai yang tersimpan pada order. Tolak jika tidak sesuai.
4. Update status order Tandai order sebagai PAID atau status yang sesuai, lalu kembalikan true.
5. Idempotency Pastikan endpoint bersifat idempotent — jika notifikasi yang sama diterima lebih dari sekali, tidak terjadi duplikasi pemrosesan order.

Contoh Implementasi Client (Java / Spring Boot)

@RestController
public class PaymentCallbackController {

    @Autowired
    private OrderRepository orderRepository;

    @Value("${payment.gateway.expected-domain}")
    private String expectedDomain;

    @PostMapping("/notify")
    public Boolean notify(
            @RequestParam String transactionId,
            @RequestParam Double amount,
            @RequestHeader("domain") String domain) throws TransactionNotFound {

        // 1. Verifikasi domain
        if (!expectedDomain.equals(domain)) {
            return false;
        }

        // 2. Cari transaksi
        Order order = orderRepository.findById(transactionId)
                .orElseThrow(() -> new TransactionNotFound(
                        "Transaction not found: " + transactionId));

        // 3. Validasi amount
        if (!order.getAmount().equals(amount)) {
            log.warn("Amount mismatch for transactionId={}: expected={}, received={}",
                    transactionId, order.getAmount(), amount);
            return false;
        }

        // 4. Idempotency — jangan proses ulang jika sudah PAID
        if (order.getStatus() == OrderStatus.PAID) {
            return true;
        }

        // 5. Update status order
        order.setStatus(OrderStatus.PAID);
        order.setPaidAt(LocalDateTime.now());
        orderRepository.save(order);

        return true;
    }
}

Contoh Implementasi Client (JavaScript / Express)

const express = require('express');
const app = express();
app.use(express.urlencoded({ extended: true }));

app.post('/notify', async (req, res) => {
  const { transactionId, amount } = req.body;
  const domain = req.headers['domain'];

  // 1. Verifikasi domain
  if (domain !== process.env.EXPECTED_DOMAIN) {
    return res.status(200).send('false');
  }

  try {
    // 2. Cari transaksi
    const order = await Order.findById(transactionId);
    if (!order) {
      return res.status(200).send('false'); // TransactionNotFound
    }

    // 3. Validasi amount
    if (order.amount !== parseFloat(amount)) {
      console.warn(`Amount mismatch: expected ${order.amount}, got ${amount}`);
      return res.status(200).send('false');
    }

    // 4. Idempotency — jangan proses ulang jika sudah PAID
    if (order.status === 'PAID') {
      return res.status(200).send('true');
    }

    // 5. Update status order
    await Order.updateOne(
      { _id: transactionId },
      { status: 'PAID', paidAt: new Date() }
    );

    return res.status(200).send('true');

  } catch (err) {
    console.error('Notification error:', err.message);
    return res.status(200).send('false');
  }
});

app.listen(3000);
